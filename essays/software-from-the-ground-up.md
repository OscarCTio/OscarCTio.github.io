<div class="essay-section foundation">
  <h2>Starting with the Basics</h2>

  <p>
    Through Python, I learned foundational programming concepts such as variables, functions, conditionals, loops, and debugging. These ideas may seem basic now, but they were the starting point for understanding how software can be used to automate tasks and solve real problems. Learning Python marked my first real exposure to software engineering and helped shape the way I approach problem-solving today.
  </p>
</div>

<div class="essay-section growth">
  <h2>Building Confidence Through Projects</h2>

  <p>
    As I progressed through my coursework, I began applying these foundational skills to different assignments and projects. Writing code became less about memorizing syntax and more about designing logical solutions. I learned how to break complex problems into smaller parts, test my ideas, debug mistakes, and improve my solutions through trial and error.
  </p>
</div>

<div class="essay-section growth">
  <h2>Seeing Software as a Tool</h2>

  <p>
    These experiences helped me gain confidence in my ability to use software as a practical tool. Programming was no longer just something I did for assignments. It became a way to understand systems, explore ideas, and create solutions. Each project helped me see that software engineering is not only about writing code, but also about thinking clearly, organizing ideas, and building something that works reliably.
  </p>
</div>

<div class="essay-section future">
  <h2>Looking Ahead</h2>

  <p>
    Looking ahead, I hope to continue developing my software engineering skills by working on more advanced systems that connect software with hardware and real-world applications. I am especially interested in projects where programming can be used to control, measure, analyze, or improve physical systems. Ultimately, I see software engineering as a discipline of continuous learning, and I am motivated to keep growing both technically and professionally as I move forward.
  </p>
